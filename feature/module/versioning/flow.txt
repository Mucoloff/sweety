1. GitHub Release → pubblicata
2. Webhook → POST al server
3. Server:
     a) ReleaseManager scarica jar base
     b) Salva in storage/base/
     c) Preload default patch se vuoi
4. Client launcher:
     a) GET /latest
     b) Riceve versione corrente
     c) Se update:
          i) GET /download?clientId=XXX
          ii) Server:
               - patch jar in memoria (FieldInjector + Watermark)
               - serve in streaming
          iii) Launcher/app sostituisce jar locale
5. Launcher/app avvia nuova versione
